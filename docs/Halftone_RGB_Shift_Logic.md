# ハーフトーンにおけるRGBシフト（色収差）処理のロジック

`MugAdvancedHalftone.fuse` (v1.95+) では、レンズの物理特性に基づく**「放射状（Radial）RGBシフト」**を実装しています。これは古いレンズに見られる倍率色収差（Transverse Chromatic Aberration）をシミュレートし、画面の端に行くほど色が大きく滲むリアルな効果を提供します。

---

## 1. 放射状色収差の仕組み

アナログレンズでは、光の波長（色）によって屈折率が僅かに異なるため、像の拡大率に差が生じます。中心部は正しく重なりますが、周辺部に行くほど赤（R）や青（B）の輪郭が外側または内側へ分離していきます。

本エフェクトでは、画面中心（`0.5, 0.5`）からの距離と方向に比例したオフセットを各チャンネルに適用します。

- **中心部**: ズレ量は 0 となり、像は鮮明です。
- **周辺部**: 中心からの距離に比例してズレが大きくなります。
- **方向**: 中心から外側に向かうベクトル（放射状）に沿って色が分離します。

---

## 2. 座標オフセット（収差量）の計算と最適化

パフォーマンスを最大化するため、ピセル空間での距離計算（平方根など）を排除し、UV空間（0.0 〜 1.0）での線形計算のみで実装しています。

### 計算式（概念）
UV座標を `uv`、中心を `mid (0.5, 0.5)`、ユーザー指定の強度を `shiftVal` とすると：

```c
float2 uvRel = uv - mid;
float2 shiftROFF = -uvRel * shiftVal; // 赤は中心方向へ（収縮方向）
float2 shiftBOFF =  uvRel * shiftVal; // 青は外側方向へ（拡大方向）
```

この `shiftROFF/BOFF` をサンプリング座標に加算することで、色成分を分離します。

### GPU最適化
`RenderDotsKernel` および `updateBestCell` 内でこの計算は実行されます。
1. **除算・平方根の排除**: 全てをUV相対座標の乗算のみで完結させています。
2. **サンプリング間引き**: `shiftVal` が 0 の場合は、最適化パスにより1回のサンプリングで処理を終了します。

---

## 3. 単位の仕様（割合ベース）

ズレの量は「ピクセル」ではなく**「画像サイズに対する割合（Ratio）」**で指定します（UI範囲: 0.0 〜 0.1）。

- **解像度独立**: 0.01 と設定すれば、HD解像度でも4K解像度でも、視覚的に「同じ割合のズレ」が維持されます。
- **最大値の意味**: 0.1 は、画面の角（半径1.0相当）において画像サイズの 10% 分のズレが生じることを意味します。実用上の色収差としては非常に強力な値です。

---

## 4. 適用範囲

この効果は以下のサンプリング時に適用されます：
- **Use Original Color (ドットの色)**: 各ドットが拾う色自体が滲みます。
- **Blend With Input (背景合成)**: 背景の映像も同様に色収差が発生します。

これらを組み合わせることで、ドットと背景が一体となった強力なアナログ・レンズ・ルックを実現します。
